## 3. Remaining cleanup — the RIGHT long-term fixes (not band-aids)

### 3a. qyl AnomalyEndpointsTests (3 failing) — time-dependent test rot
**Root cause (verified):** test seeds at hardcoded
`TokenStart = new(2026,5,23,10,0,0,…)` (`tests/qyl.collector.tests/Functional/AnomalyEndpointsTests.cs:103`),
but the endpoint filters a **relative** window:
`ComputeCutoffNano(hours) => GetUtcNow().AddHours(-hours)`
(`services/qyl.collector/Analytics/AnomalyService.cs:208-210`), query
`WHERE start_time_unix_nano >= cutoff` (`AnomalyService.cs:79`, returns
`sampleCount:0` at line 98 when empty). Seeded 2026-05-23 data is now outside
the 24h window → `sampleCount:0`/`mean:0`. Endpoint is correct; fixture aged out.

**Right fix** — make the anchor relative (immune to clock drift). Replace the
`static readonly DateTimeOffset TokenStart = new(2026,5,23,…)` with:
```csharp
private static DateTimeOffset TokenStart
{
    get
    {
        var now = DateTimeOffset.UtcNow;
        // Top of an hour 2h ago: both seeded spans (TokenStart + 5min) land in
        // ONE hourly bucket AND inside the endpoint's relative hours=24 window.
        var topOfHour = new DateTimeOffset(now.Year, now.Month, now.Day, now.Hour, 0, 0, TimeSpan.Zero);
        return topOfHour.AddHours(-2);
    }
}
```
Then re-run; all 3 should pass (bucketing math unchanged: spans share one hour
→ sampleCount=1, mean = bucket total). Do NOT delete — buries a working endpoint.

### 3b. qyl Semconv.RegenCleanTests (1 failing) — CI-only meta-gate
**Root cause (verified):** asserts `git status --porcelain` empty BEFORE running
`nuke OtelConventions` (`tests/qyl.collector.tests/Semconv/RegenCleanTests.cs:21-27`).
On a dirty working tree (any active dev), it hits `Assert.Fail("Working tree
was dirty…")` at the pre-check — not regeneration drift. Tagged
`[Trait("Category","regen")]`.

**Right fix** — make it **skip** (not fail) when its precondition isn't met, so
local `dotnet test` stays green while CI (clean checkout) still enforces it.
xUnit v3 supports dynamic skip. In the pre-check block, replace `Assert.Fail(…)`
with:
```csharp
Assert.Skip("Regen gate is CI-only; working tree is dirty (expected during local dev).");
return;
```
(Keep the post-regen drift `Assert.Fail` — that one is the real gate.) Do NOT
delete — removes a real drift guard.
